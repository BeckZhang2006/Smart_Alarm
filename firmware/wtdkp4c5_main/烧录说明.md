# WTDKP4C5-S1 固件烧录指南

## 🔌 硬件连接

WTDKP4C5-S1 开发板使用 **USB Serial/JTAG** 接口进行烧录，只需要一根 USB-C 线连接电脑即可。

```
┌─────────────────────┐
│   WTDKP4C5-S1       │
│  ┌───────────────┐  │
│  │  ESP32-P4     │  │
│  │  (主控)       │  │
│  │              ═╪══╪═══ USB-C 连接到电脑
│  └───────────────┘  │      (数据和供电)
│  ┌───────────────┐  │
│  │  ESP32-C5     │  │
│  │  (WiFi/CSI)   │  │
│  └───────────────┘  │
└─────────────────────┘
```

## 📝 烧录前准备

### 1. 安装 USB 驱动

首次使用需要安装 USB 驱动：

**方法一：使用 ESP-IDF 自带驱动**
- 运行 `D:\Espressif\tools\idf-driver\idf-driver-esp-usb-jtag-0.1.0.exe`（如果存在）

**方法二：手动安装**
1. 下载 ESP32-P4 USB 驱动：https://docs.espressif.com/projects/esp-idf/en/latest/esp32p4/api-guides/usb-serial-jtag.html
2. 或让 Windows 自动搜索安装（连接板子后会提示）

### 2. 进入下载模式

**步骤：**
1. 按住板子上的 **BOOT** 按键
2. 按一下 **RST** 复位按键
3. 松开 **BOOT** 按键
4. 此时板子进入下载模式，等待烧录

## 🚀 烧录方法

### 方法一：使用一键烧录脚本（推荐）

提供了两个烧录脚本：

| 脚本 | 说明 |
|------|------|
| `flash.bat` | **完整烧录** - 先擦除 Flash，再烧录（会清除所有保存的设置） |
| `flash_without_erase.bat` | **快速烧录** - 保留 NVS 数据（闹钟设置等） |

**操作步骤：**
1. 双击 `flash.bat` 或 `flash_without_erase.bat`
2. 按提示进入下载模式（按住 BOOT → 按 RST → 松开 BOOT）
3. 按任意键开始烧录
4. 等待烧录完成

### 方法二：使用命令行手动烧录

```powershell
# 设置工具路径
$ESPTOOL = "D:\Espressif\python_env\idf5.5_py3.11_env\Scripts\python.exe"
$ESPTOOL_PY = "D:\Espressif\frameworks\esp-idf-v5.5.2\components\esptool_py\esptool\esptool.py"

# 进入下载模式后执行
& $ESPTOOL $ESPTOOL_PY --chip esp32p4 --port COMx --baud 921600 erase_flash

& $ESPTOOL $ESPTOOL_PY --chip esp32p4 --port COMx --baud 921600 `
    --before default_reset --after hard_reset write_flash -z `
    --flash_mode dio --flash_freq 40m --flash_size 2MB `
    0x2000 "build/bootloader/bootloader.bin" `
    0x8000 "build/partition_table/partition-table.bin" `
    0x10000 "build/smart_alarm_main.bin"
```

> 将 `COMx` 替换为实际的串口号（如 COM3、COM4）

### 方法三：使用 ESP-IDF 工具

如果已安装 ESP-IDF 环境：

```powershell
# 进入项目目录
cd d:\workspace\alarm\smart_alarm\firmware\wtdkp4c5_main

# 先进入下载模式，然后执行
python %IDF_PATH%\components\esptool_py\esptool\esptool.py erase_flash

# 或直接烧录
python %IDF_PATH%\components\esptool_py\esptool\esptool.py write_flash @build\flash_args
```

## 🔍 查看串口号

### 方法 1：设备管理器
1. 右键"此电脑" → 管理 → 设备管理器
2. 展开"端口 (COM 和 LPT)"
3. 查找 "USB Serial/JTAG Unit" 或类似设备

### 方法 2：PowerShell 命令
```powershell
# 列出所有串口
[System.IO.Ports.SerialPort]::GetPortNames()

# 或使用 WMI
Get-WmiObject Win32_SerialPort | Select-Object DeviceID,Description
```

### 方法 3：插拔法
1. 先执行 `Get-WmiObject Win32_SerialPort` 记录当前端口
2. 插入开发板
3. 再次执行命令，对比新增的端口

## 📋 烧录文件说明

| 文件 | 地址 | 大小 | 说明 |
|------|------|------|------|
| `bootloader/bootloader.bin` | 0x2000 | ~23 KB | 引导程序 |
| `partition_table/partition-table.bin` | 0x8000 | ~3 KB | 分区表 |
| `smart_alarm_main.bin` | 0x10000 | ~773 KB | 主固件 |

## ⚠️ 常见问题

### 1. 找不到串口
- 检查 USB 线是否支持数据传输（有些只支持充电）
- 更换 USB 端口尝试
- 检查驱动是否正确安装
- 查看设备管理器中是否有未知设备

### 2. 烧录失败/超时
- 确认已进入下载模式（按住 BOOT + 按 RST）
- 检查波特率是否过高，可尝试降低为 460800
- 检查串口号是否正确
- 确保没有其他程序占用该串口

### 3. 烧录后无法启动
- 检查是否选择了正确的芯片型号（esp32p4）
- 检查 Flash 参数是否正确（DIO, 40MHz, 2MB）
- 尝试完整擦除后重新烧录

### 4. 自动烧录失败
如果 `flash.bat` 的 `auto` 端口检测失败，请手动修改脚本中的 `PORT` 变量：
```bat
set "PORT=COM3"  :: 修改为你的实际串口号
```

## ✅ 烧录成功标志

烧录完成后会看到类似信息：
```
Hard resetting via RTS pin...
Done
```

然后设备会自动重启，串口监视器会输出启动日志。

## 🌐 首次使用

烧录成功后：
1. 设备创建 WiFi 热点：`MyESP32P4_AP`
2. 密码：`12345678`
3. 用手机/电脑连接该热点
4. 浏览器访问：`http://192.168.4.1`
5. 即可配置闹钟和查看状态
